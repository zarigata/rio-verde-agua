Metodologia

Objetivo Geral

Desenvolver uma plataforma computacional de analise e simulacao do sistema de abastecimento de agua do municipio de Rio Verde, GO, utilizando dados oficiais para apoiar a gestao de recursos hidricos municipais.

Objetivos Especificos

  1. Mapear e catalogar a infraestrutura hidrica municipal com base em fontes oficiais (Saneago, Camara Municipal, IBGE);
  2. Implementar motor de simulacao de transferencia hídrica entre pontos de reservacao e distribuicao;
  3. Desenvolver sistema de recomendacao baseado em regras hidrologicas (ABNT NBR 12211 e NBR 12218);
  4. Criar interface interativa para visualizacao geografica e analise de cenarios operacionais;
  5. Classificar todos os dados utilizados quanto a procedencia, nivel de certeza e data de referencia;
  6. Simular cenarios de estresse hidrico (seca, crescimento populacional, falhas operacionais) e avaliar vulnerabilidades do sistema.

Abordagem Metodologica

Trata-se de uma pesquisa aplicada, com abordagem quantitativa, desenvolvida em tres etapas:

Etapa 1 — Levantamento de Dados: Coleta de dados oficiais junto a Saneago (relatorio institucional, agosto 2023), Camara Municipal de Rio Verde (materia sobre o sistema Rio Verdinho, outubro 2024) e IBGE (estimativa populacional 2025).

Etapa 2 — Modelagem Computacional: Implementacao de formulas hidrologicas (demanda, producao efetiva, balanco hidrico, indice de estresse) e motor de simulacao de transferencia entre pontos.

Etapa 3 — Desenvolvimento da Plataforma: Criacao de interface web com mapa interativo (Leaflet), graficos analíticos (Chart.js) e simulador de cenarios, todos executaveis diretamente no navegador.

Formulas Utilizadas

Demanda Diaria

D = P x C x Fi / 1000

Onde: P = populacao, C = consumo per capita (L/hab/dia), Fi = fator industrial, resultado em m3/dia.

Base: ABNT NBR 12211

Producao Efetiva

Pef = Pbruta x (1 - perda%)

Onde: Pbruta = capacidade nominal da ETA, perda% = perdas na distribuicao (adotado 25%).

Base: ABNT NBR 12218

Indice de Estresse Hidrico

IEH = D / Pef

Onde: D = demanda, Pef = producao efetiva. IEH < 0,1 = sem estresse; IEH > 0,4 = alto estresse.

Adaptado de indicadores da ONU

Autonomia do Reservatorio

A = V / Cmedio

Onde: V = volume disponivel (m3), Cmedio = consumo medio diario (m3/dia). Resultado em dias.

Formula padrao de operacao de reservatorios

Transferencia Viavel

Vdisp = max(0, Vatual - Rmin)

Onde: Vdisp = volume disponivel para cessao, Rmin = reserva minima obrigatoria.

Regra operacional do simulador

Volume Recebido no Destino

Vrec = Vtransf x (1 - perdaop)

Onde: Vtransf = volume transferido, perdaop = perda operacional na transferencia.

Classificacao de Dados

Todos os dados utilizados sao classificados quanto a certeza para garantir transparencia.

Confirmado

Dado obtido de fonte oficial e verificavel. Ex.: capacidade da ETA Abobora (Saneago, 2023).

Derivado

Estimado a partir de dado real por calculo direto. Ex.: capacidade em m3/dia a partir de L/s.

Aproximado

Estimado com baixa precisao ou sem fonte direta. Ex.: coordenadas por geocodificacao de bairro.

Limitacoes

  • Coordenadas: Obtidas por geocodificacao aproximada, nao confirmadas por levantamento GPS.
  • Consumo por zona: Estimado com base na populacao atendida e consumo per capita ABNT, nao por medicao direta.
  • Volume atual dos reservatorios: Dados operacionais nao publicados pela Saneago; valores simulados para demonstracao.
  • Historico temporal: Nao foi possivel obter series historicas de consumo ou producao; o simulador usa parametros medios.
  • AMAE: A pagina de analise de impacto da AMAE retornou erro 404; dados reguladores nao foram acessados.
